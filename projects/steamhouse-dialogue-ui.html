Steam House — Dialogue System & UI/UX

Unreal Engine Blueprint Behavior Trees Blackboard UMG UI/UX

Interactive adaptation inspired by Jules Verne’s La Maison à vapeur. I designed the dialogue architecture (Behavior Tree-driven), the typewriter presentation system, and implemented the UI + menu interactions across the game.

Dialogue Demo

NPC interaction → dialogue appears with typewriter effect → advance/skip → state updates (blackboard/quest).

Overview

The challenge was turning a literary narrative into interactive dialogue that remains scalable. Instead of hardcoding lines in widgets, conversations are authored as Behavior Trees and rendered by a UI layer focused on readability, pacing, and feedback.

My Role

  • Dialogue system design + implementation (Behavior Tree pipeline)
  • Typewriter effect + skip/advance logic
  • Dialogue UI (choices, transitions, animations)
  • Main menu + pause menu implementation (mouse & keyboard)
  • Quest/blackboard updates triggered by dialogue

Goals

  • Modular dialogues per NPC (easy to extend)
  • Branching + choices + quest gating
  • Readable cinematic UI with consistent timing
  • Clear interaction rules across dialogue + menus

Tools

Unreal Engine Blueprint Behavior Tree Blackboard UMG UI Animations

Dialogue System Architecture (Behavior Tree)

Each NPC has a corresponding Behavior Tree. Blackboard conditions gate branches (story state, quest state, choice results), and BT tasks trigger the UI layer (run line, show choices, start/end quest beats).

Interaction Flow

  • Interact → identify NPC → load its Behavior Tree
  • BT selects the correct line/branch using blackboard state
  • UI widget receives text + speaker context
  • Typewriter renders the line (skip/advance)
  • On completion → BT continues or ends the dialogue

Blueprint Implementation (Proof)

Event-driven wiring: create widget, add to viewport, and bind “Text Finished” to continue the flow. Dialogue can also update quest rewards and blackboard variables directly from the conversation.

Menus & Interaction UX (Same Project)

I implemented the full menu UX across the game (mouse & keyboard): main menu and in-game pause menu. The goal was consistent interaction rules, readable layout hierarchy, and animation feedback that matches the game’s pacing.

UX Rules

  • Consistent confirm/back behavior across UI screens
  • Clear hover states + click feedback (visual + timing)
  • Dialogue locks movement; pause menu freezes gameplay and redirects input to UI
  • Transitions (fade/slide) use consistent timing for a cohesive feel

Result

  • Modular NPC dialogue authored as Behavior Trees (easy to extend)
  • State-driven branching via blackboard variables

  • Complete UI implementation across the project: dialogue + menus
